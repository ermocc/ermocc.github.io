监听者设计模式

二默 12/18/2021 java设计模式

设计模式是前人总结出的一些代码设计风格，在代码开发中使用设计模式可以降低代码耦合，使代码更加灵活。

监听者模式主要包含三个角色：

  • 事件对象（具体的处理信息）
  • 事件监听器（一般为接口，针对事件的具体处理行为）
  • 事件源（监听器的管理，比如订阅、删除、添加等）

接下来简单实现一个监听者模式。

针对手机 App，我们有打开、关闭、切入、切出等不同操作行为，现在就拿这个例子进行编码，对不同的手机 App 行为进行监听。

首先是 App 事件对象。

public class AppEvent extends EventObject {

    public AppEvent(Object source) {
        super(source);
    }

    public void doEvent() {
        System.out.println("Notify event source:" + this.getSource());
    }
}

具体的事件对象 source。

@Data
public class AppEventMsg {
    private String eventType;
    private String eventName;
    private String unionId;
    private Date createDate;
}

事件监听器。

public interface AppEventListener {
    void handleEvent(EventObject event);
}

具体的事件监听器实现，这里以打开 App 和关闭 App 举例。

public class OpenAppEventListener implements AppEventListener {

    @Override
    public void handleEvent(EventObject event) {
        System.out.println("Record open app event!");
    }
}

public class CloseAppEventListener implements AppEventListener {

    @Override
    public void handleEvent(EventObject event) {
        System.out.println("Record close app event!");
    }
}

事件源，对事件监听器进行管理。

public class AppEventSource {

    private final ConcurrentHashMap<String, AppEventListener> listenerMap = new ConcurrentHashMap<>();

    public void addListener(String eventType, AppEventListener listener) {
        listenerMap.put(eventType, listener);
    };

    public void removeListener(String eventType) {
        listenerMap.remove(eventType);
    }

    public void notifyListener(AppEvent eventObject) {
        eventObject.doEvent();
        AppEventMsg source = (AppEventMsg) eventObject.getSource();
        AppEventListener listener = listenerMap.get(source.getEventType());
        if (listener != null) {
            listener.handleEvent(eventObject);
        }
    }

}

测试

public class AppEventTest {

    public static void main(String[] args) {
        AppEventSource appEventSource = new AppEventSource();
        appEventSource.addListener("openApp", new OpenAppEventListener());
        appEventSource.addListener("closeApp", new CloseAppEventListener());

        AppEventMsg appEventMsg = new AppEventMsg();
        appEventMsg.setEventType("openApp");
        appEventMsg.setCreateDate(new Date());
        appEventMsg.setEventName("openApp");
        appEventMsg.setUnionId(UUID.randomUUID().toString().replaceAll("-", ""));
        AppEvent eventObject = new AppEvent(appEventMsg);
        appEventSource.notifyListener(eventObject);

        AppEventMsg closeAppMsg = new AppEventMsg();
        closeAppMsg.setEventType("closeApp");
        closeAppMsg.setCreateDate(new Date());
        closeAppMsg.setEventName("closeApp");
        closeAppMsg.setUnionId(UUID.randomUUID().toString().replaceAll("-", ""));
        AppEvent closeEventObject = new AppEvent(closeAppMsg);
        appEventSource.notifyListener(closeEventObject);
    }
}

监听者模式是开发过程中最常用的设计模式之一，熟悉监听者模式对开发和开源框架源码阅读都有很大的帮助。
